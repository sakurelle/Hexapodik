Измени существующий проект ServoTester. Преврати его в контроллер одной ноги гексапода с тремя сервоприводами.

Используй существующий стек:

- Visual Studio Code;
- PlatformIO;
- framework = espidf;
- обычный ESP32-WROOM-32 / esp32dev;
- C++;
- ESP-IDF;
- LEDC;
- FreeRTOS.

Не используй:

- Arduino Framework;
- Arduino.h;
- библиотеку Servo;
- setup();
- loop();
- внешние библиотеки.

Не создавай новый проект рядом. Измени существующий проект ServoTester.

Назначение
==========

Проект должен управлять одной ногой гексапода, состоящей из трёх суставов:

1. Coxa — поворот ноги относительно корпуса.
2. Femur — подъём и опускание бедра.
3. Tibia — сгибание нижней части ноги.

Управление должно выполняться через Serial Monitor командами с компьютера.

Подключение серв
================

Используй следующие GPIO:

- Tibia: GPIO25;
- Femur: GPIO26;
- Coxa: GPIO27.

Все три сервопривода должны управляться отдельными каналами LEDC.

Частота PWM:

- 50 Гц;
- период 20 мс.

Сервоприводы должны питаться от отдельного источника 5–6 В.

Обязательно указать в README:

- не питать три сервопривода от ESP32;
- объединить GND источника серв и GND ESP32;
- поставить электролитический конденсатор не менее 1000–2200 мкФ около разъёмов серв;
- желательно поставить по 100 нФ около каждого разъёма;
- источник питания должен выдерживать пусковой ток сразу трёх серв.

Калибровка
==========

Используй индивидуальную трёхточечную калибровку.

Порядок значений:

- первое значение соответствует углу −45°;
- второе значение соответствует углу 0°;
- третье значение соответствует углу +45°.

Tibia, GPIO25:

- −45° = 2017 мкс;
- 0° = 1498 мкс;
- +45° = 969 мкс.

Femur, GPIO26:

- −45° = 1976 мкс;
- 0° = 1462 мкс;
- +45° = 934 мкс.

Coxa, GPIO27:

- −45° = 1995 мкс;
- 0° = 1491 мкс;
- +45° = 961 мкс.

У всех трёх серв направление инвертировано:

- при увеличении угла длительность импульса уменьшается.

Не заменяй индивидуальную калибровку общим диапазоном 1000–2000 мкс.

Создай структуру наподобие:

struct ServoCalibration {
    const char* name;
    gpio_num_t gpio;
    ledc_channel_t channel;

    int32_t pulse_minus_45_us;
    int32_t pulse_zero_us;
    int32_t pulse_plus_45_us;

    float current_angle_deg;
    float target_angle_deg;
};

Для каждой сервы должен существовать отдельный экземпляр этой структуры.

Пересчёт угла в импульс
=======================

Реализуй функцию:

int32_t angle_to_pulse_us(
    const ServoCalibration& servo,
    float angle_deg
);

Используй кусочно-линейную интерполяцию по трём измеренным точкам.

Для диапазона от −45° до 0°:

- интерполировать между pulse_minus_45_us и pulse_zero_us.

Для диапазона от 0° до +45°:

- интерполировать между pulse_zero_us и pulse_plus_45_us.

Не рассчитывай центр как среднее крайних точек. Используй фактически измеренное значение центра.

Обязательно ограничивать командный угол диапазоном:

- минимум −45°;
- максимум +45°.

Функция должна корректно работать с дробными углами, например:

- −12.5°;
- 0.0°;
- 27.3°.

Перед записью PWM ограничивать итоговый импульс минимальным и максимальным калибровочным значением конкретной сервы.

LEDC
====

Настрой один LEDC timer:

- LEDC low-speed mode;
- частота 50 Гц;
- подходящая разрядность, предпочтительно 16 бит;
- автоматический источник тактирования, совместимый с ESP32.

Используй три LEDC channel:

- отдельный канал для Tibia;
- отдельный канал для Femur;
- отдельный канал для Coxa.

Реализуй:

esp_err_t servo_pwm_init();
esp_err_t servo_write_pulse_us(
    const ServoCalibration& servo,
    int32_t pulse_us
);
esp_err_t servo_write_angle(
    ServoCalibration& servo,
    float angle_deg
);

Преобразование микросекунд в duty cycle выполняй через uint64_t, чтобы избежать переполнения и потери точности.

Сервы должны получать PWM непрерывно после запуска.

Начальное положение
===================

При запуске не допускай резкого неконтролируемого перехода.

Начальная поза:

- Coxa = 0°;
- Femur = 0°;
- Tibia = 0°.

После инициализации PWM плавно привести все сервы к начальной позе за 1500 мс.

Плавное движение
================

Нельзя мгновенно перескакивать между удалёнными положениями.

Создай отдельную FreeRTOS-задачу управления с периодом 20 мс.

Она должна:

1. Хранить текущий угол каждого сустава.
2. Хранить целевой угол каждого сустава.
3. Каждые 20 мс приближать текущий угол к целевому.
4. Ограничивать максимальную скорость движения.

Создай константу:

constexpr float DEFAULT_SPEED_DEG_PER_SEC = 45.0f;

Максимальное изменение за один цикл:

speed_deg_per_sec * 0.020f.

Реализуй возможность задавать скорость команды.

Не использовать блокирующие циклы с delay при обычном управлении.

Команды Serial
==============

Настрой UART0 / стандартный Serial Monitor:

- скорость 115200;
- команды завершаются Enter;
- поддерживать CR, LF и CRLF;
- не использовать Arduino Serial;
- использовать штатный UART-драйвер ESP-IDF или stdin через консоль ESP-IDF.

Реализуй следующие команды.

1. Установить все три угла:

leg <coxa> <femur> <tibia>

Пример:

leg 10 -20 30

Команда устанавливает:

- Coxa = 10°;
- Femur = −20°;
- Tibia = 30°.

2. Установить один сустав:

coxa <angle>
femur <angle>
tibia <angle>

Примеры:

coxa 15
femur -10
tibia 25

3. Установить скорость:

speed <degrees_per_second>

Пример:

speed 30

Разрешённый диапазон скорости:

- минимум 5°/с;
- максимум 180°/с.

4. Вернуться в нулевую позу:

center

Эквивалент:

leg 0 0 0

5. Показать текущее состояние:

status

Выводить:

- текущий угол;
- целевой угол;
- текущий рассчитанный импульс;
- GPIO;
- калибровочные значения.

6. Остановить движение в текущей позиции:

stop

Команда должна приравнять целевые углы текущим углам. PWM при этом продолжает удерживать сервы.

7. Выполнить тест каждого сустава:

test coxa
test femur
test tibia

Последовательность теста:

- 0°;
- −20°;
- +20°;
- 0°.

Двигаться плавно с текущей установленной скоростью.

8. Выполнить тест всей ноги:

demo

Без обратной кинематики выполнить безопасную последовательность:

Поза 1:
- Coxa = 0°;
- Femur = 0°;
- Tibia = 0°.

Поза 2:
- Coxa = −15°;
- Femur = −15°;
- Tibia = +20°.

Поза 3:
- Coxa = +15°;
- Femur = −15°;
- Tibia = +20°.

Поза 4:
- Coxa = 0°;
- Femur = 0°;
- Tibia = 0°.

Между позами ждать завершения плавного движения, но не блокировать задачу PWM.

9. Показать справку:

help

Парсер команд
=============

Парсер должен:

- проверять количество аргументов;
- проверять преобразование строк в числа;
- отклонять NaN и бесконечные значения;
- ограничивать углы диапазоном −45…+45°;
- сообщать, если введённый угол был ограничен;
- не аварийно завершаться при неправильной команде;
- не использовать небезопасные функции без проверки размера буфера.

Максимальная длина команды — 128 символов.

При ошибке выводить понятный пример правильного синтаксиса.

Безопасность
============

Добавь программные ограничения:

constexpr float SERVO_MIN_ANGLE_DEG = -45.0f;
constexpr float SERVO_MAX_ANGLE_DEG = 45.0f;

Запрещено отправлять импульсы за пределами индивидуальных калибровочных значений.

При ошибке LEDC:

- вывести сообщение через ESP_LOGE;
- не продолжать запуск задачи движения;
- не отправлять случайные значения на GPIO.

Добавь команду:

disable

Она должна отключить выход PWM для трёх серв, чтобы они перестали удерживать положение.

Добавь команду:

enable

Она должна:

1. Включить PWM.
2. Начать с импульсов, соответствующих текущим сохранённым углам.
3. Не делать резкий переход через произвольную позицию.

После `disable` внутренние текущие и целевые углы должны сохраняться.

Синхронизация
=============

Задача консоли и задача движения работают параллельно.

Защити общие значения:

- target_angle_deg;
- current_angle_deg;
- установленную скорость;
- флаг enabled.

Используй mutex или другой корректный механизм FreeRTOS.

Не удерживай mutex во время длительных ожиданий и вывода больших объёмов текста.

PlatformIO
==========

Сохрани проект как PlatformIO ESP-IDF.

platformio.ini должен содержать рабочее окружение:

[platformio]
default_envs = esp32dev
src_dir = main

[env:esp32dev]
platform = platformio/espressif32
board = esp32dev
framework = espidf
monitor_speed = 115200
upload_speed = 115200

Не переходить на Arduino Framework.

Проверь CMakeLists.txt и main/CMakeLists.txt.

main/CMakeLists.txt должен включать main.cpp.

Проверка
========

После изменения проекта выполни:

1. pio project config
2. pio run -e esp32dev
3. pio device list

Исправляй ошибки до успешной сборки.

Если ESP32 подключена, выполни прошивку:

pio run -e esp32dev -t upload

После прошивки открой монитор:

pio device monitor --baud 115200

Проверь хотя бы команды:

help
status
center
leg 0 0 0

Не запускай автоматически большие углы при первой загрузке.

Не утверждай, что физические движения проверены, если ты не можешь наблюдать подключённую ногу.
